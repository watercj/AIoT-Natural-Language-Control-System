# Overview

This project focuses on the development of an **AIoT Natural Language Control System** that bridges the gap between human natural language and hardware execution. By integrating a **Natural Language Processing (NLP)** translation engine with an **Arduino(ESP32-based hardware framework)**, the system allows non-technical users to control complex IoT devices using intuitive, conversational commands (e.g., "Flash the red light if an object is too close"). 

Beyond accessibility, the system prioritizes data integrity and privacy by implementing local **RSA encryption**, ensuring that all environmental telemetry remains secure from the edge to the cloud.

# Web

The system features an intuitive web-based dashboard that serves as the primary bridge for human-computer interaction :

* **Natural Language Input Field:** Users can directly input commands in Chinese (e.g., "Wait for one second, then turn on the red light").
* **Real-Time Status Monitoring:** Leverages the **Firebase API** to provide live visualizations of sensor data (temperature, humidity, distance, and ambient light) transmitted from the ESP32 edge node.
* **Command Translation Preview:** The interface provides a real-time preview of the logical code generated by the NLP engine (e.g., `(do LCDS 2)`). This ensures transparency and enhances the system's overall debuggability.

# Arduino

This ESP32-based development board operates as an intelligent edge computing node that integrates multi-sensor data acquisition with a sophisticated command parsing engine. The system is designed to bridge local hardware interactions with cloud-based intelligence, implementing a secure and responsive IoT architecture.

### 1. NLP Translation Engine (syntax_parser.h)
The heart of the system is a logic parser that translates semantic intent into precise hardware execution codes. It supports three hierarchical command types:

* **Simple Statement**: Direct control of a single hardware module.
    * *Natural Language:* "Sound the buzzer."
    * *System Translation:* `(do BUZ)`
* **Compound Statement**: Sequential or logic-linked actions.
    * *Natural Language:* "Wait for 1 second, then turn on the red LED for 2 seconds."
    * *System Translation:* `((sleep 1) (do LEDL OER T 2))`
* **Conditional Statement**: Sensor-driven automation logic.
    * *Natural Language:* "If the distance is between 100 and 120, blink the yellow LED twice."
    * *System Translation:* `(if (cond SSC >= 100 and SSC <= 120) then (do LEDS OEY C 2))`

### 2. Hardware and Sensing Integration (ESP32 Edge Node)
The system utilizes the ESP32 for high-performance edge computing and multi-modal feedback:
* **Data Acquisition**: Real-time monitoring via DHT11 (Temp/Humidity), Ultrasonic (Distance), and LDR (Ambient Light) sensors.
* **Feedback Control**:
    * **Visual**: An OLED display for status updates and an RGY LED for programmable patterns (Red, Green, Yellow).
    * **Auditory**: A buzzer for immediate local alerts.

### 3. Security and Connectivity
* **Cryptographic Security**: All collected data is encrypted locally using the **RSA algorithm** before transmission, preventing unauthorized access during the transit phase.
* **Dual-Protocol Cloud Sync**:
    * **Firebase Realtime Database (API)**: For remote synchronization and persistent data logging.
    * **MQTT**: For low-latency messaging and real-time command-and-control.
